## Was heisst CI & CD?

* **CD** – Continuous Delivery oder Continuous Deployment

## Was ist CI & CD?


--s--
## Continuous Delivery oder Continuous Deployment

* Beides meint in etwa das selbe
* beim Delivery gibt es aber meist noch einen manuellen Schritt um das Produkt auf Production zu deployen.

--s--
## CI


* Zwischen Entwicklung und Auslieferung wenig Reibung
* Standardisiserte Abläufe
* Schnelle Release-Zyklen

--s--

## Mögliche Strategie

* Jedes Feature oder jeder Bugfix bekommt eine User-Story (Management)
* Programmierung erfolgt je Story
* Wenn Story (auf Testsystem) Abgenommen (auf Production) ausliefern

## Konfiguration

Neue Datei in Root `.travis.yml`

--s--

# Umgebung aufbauen

```yaml

language: node_js
node_js: stable
```
* [mögliche Sprachen](https://docs.travis-ci.com/user/languages/)


--s--

# Packages Installieren

```yaml
install:
  - npm install
```

oder auch

```yaml
install:
  - npm ci
```

--s--
# Arbeit machen


```yaml

script:
  - npm run generate
```
--s--
# Deploy

```yaml

deploy:
  provider: pages
  local-dir: ./dist
  skip-cleanup: true
  github-token: $GITHUB_TOKEN
  on:
    branch: master
```

* [mögliche Deployments](https://docs.travis-ci.com/user/deployment/)
* [GitHub token](https://github.com/settings/tokens)


--s--
## Konfiguration

```yaml
language: node_js
node_js: stable
install:
  - npm ci
script:
  - npm run generate
deploy:
  provider: pages
  local-dir: ./dist
  skip-cleanup: true
  github-token: $GITHUB_TOKEN
  on:
    branch: master
```
